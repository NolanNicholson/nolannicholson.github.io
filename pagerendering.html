Static Website Compilation

Last time, I discussed some of the challenges of rendering code in a modern-looking webpage, using helpful features like syntax highlighting. The most immediately available solution, a JavaScript library from Google, worked well, but it required a server call to Google and some JavaScript execution for a result that we usually expect to be static. As implemented, it also required us to put our code in the HTML page itself, which makes it difficult to develop and test the code.

This, it turns out, is symptomatic of a broader issue: there is a lot of "work" that is really useful to be able to do automatically on HTML. Formatting code is just one example: it's also really useful to have things like page templating, so that much-reused components like headers and navigation bars don't have to be repeatedly copied and pasted.

So, I've just finished reorganizing my website to use Jinja2 for automatic template compiling, and to perform some other automatic formatting (namely, syntax highlighting) through Python in the process. Jinja2 is a templating engine that dynamically generates HTML. It is usually used from within Flask to serve websites dynamically. I invoke Jinja2 through Flask, too, but not by running a Flask app! Instead, I just use Flask to create a context for templating.

Below is the script that performs the templating. All of the page templates are stored in the library templates. They all extend a single template, base.html. Syntax highlighting is then performed using the Python library Pygments. Here is the full script:

import os
from flask import Flask, render_template
from pygments import highlight
from pygments.lexers import get_lexer_for_filename
from pygments.formatters import HtmlFormatter

app = Flask(__name__)

def code_snippet(snippet_filename, n_start=1, n_end=0):
    f = open('templates/code_snippets/' + snippet_filename, 'r')
    lines = f.read().split('\n')
    n_end = len(lines) if n_end == 0 else n_end
    text = '\n'.join(lines[n_start-1:n_end])

    hl_fmt = HtmlFormatter(linenos="table", linenostart=n_start)
    highlit_text = highlight(text,
            get_lexer_for_filename(snippet_filename),
            hl_fmt)
    return "<div class='codeholder'>\n" + highlit_text + "\n</div>"

app.jinja_env.globals.update(code_snippet=code_snippet)

with app.app_context():
    # Render syntax highlighting template
    f = open("highlighting.css", 'w')
    f.write(HtmlFormatter().get_style_defs('.highlight'))
    f.close()

    # Render individual pages
    for filename in os.listdir('templates'):
        if '.html' in filename and filename != 'base.html':
            print("Compiling template {}...".format(filename))

            # add the link back home 
            link_home = filename != 'index.html'

            rendered_html = render_template(
                    filename, link_home=link_home)
            f = open(filename, 'w')
            f.write(rendered_html)
            f.close()

The result is quite maintainable, while still allowing a huge degree of freedom (you can still just specify a page's HTML completely from scratch with no templating, if you want to), and it performs all of the "busy work" just once - instead of delegating it to JavaScript on the reader's side, on every single page load.
