Static Website Compilation

Last time, I discussed some of the challenges of rendering code in a modern-looking webpage, using helpful features like syntax highlighting. The most immediately available solution, a JavaScript library from Google, worked well, but it required a server call to Google and some JavaScript execution for a result that we usually expect to be static. As implemented, it also required us to put our code in the HTML page itself, which makes it difficult to develop and test the code.

This, it turns out, is symptomatic of a broader issue: there is a lot of "work" that is really useful to be able to do automatically on HTML. Formatting code is just one example: it's also really useful to have things like page templating, so that much-reused components like headers and navigation bars don't have to be repeatedly copied and pasted.

So, I've just finished reorganizing my website to use Jinja2 for automatic template compiling, and to perform some other automatic formatting (namely, syntax highlighting) through Python in the process. Jinja2 is a templating engine that dynamically generates HTML. It is usually used from within Flask to serve websites dynamically. I invoke Jinja2 through Flask, too, but not by running a Flask app! Instead, I just use Flask to create a context for templating.
